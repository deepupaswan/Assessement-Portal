## 🚀 Quick Start

### 1. Prerequisites
```powershell
# Verify Docker is installed
docker --version
docker compose version

# Verify PowerShell 7+
$PSVersionTable.PSVersion

# Change to project directory
cd C:\Dev
```

### 2. Build Images
```powershell
./docker-build-all.ps1
# Takes 5-15 minutes on first build (caches subsequent builds)
```

### 3. Start Services
```powershell
./docker-up.ps1
# Waits for all services to become healthy (~40 seconds)
```

### 4. Verify Services
```powershell
./docker-status.ps1
# Shows all 13 containers and their health status
```

### 5. Access Services
```
• Frontend:               http://localhost:4200
• API Gateway:           http://localhost:7080
• RabbitMQ Management:   http://localhost:15672 (guest/guest)
• Identity Service:      http://localhost:7140/health
• Answer Service:        http://localhost:7141/health
• Assessment Service:    http://localhost:7142/health
• Candidate Service:     http://localhost:7143/health
• Result Service:        http://localhost:7144/health
```

### 6. View Logs
```powershell
./docker-logs.ps1 -Service api-gateway -Follow
./docker-logs.ps1 -Service identity-service -Tail 50
./docker-logs.ps1  # All logs
```

### 7. Stop Services
```powershell
./docker-down.ps1
# Remove data as well
./docker-down.ps1 -RemoveVolumes
```

## 📞 Support & Troubleshooting

### Check Health of Services
```powershell
./docker-status.ps1
```

### View Specific Service Logs
```powershell
./docker-logs.ps1 -Service api-gateway -Follow
```

### Test Database Connectivity
```powershell
docker exec sqlserver /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P SqlPassword123! -Q "SELECT 1"
```

### Test Service Communication
```powershell
docker exec api-gateway curl http://identity-service/health
```

### Common Issues

**Container won't start:**
- Check logs: `./docker-logs.ps1 -Service <service>`
- Verify ports aren't in use: `Get-NetTCPConnection -LocalPort <port>`
- Check .env variables are set correctly

**Database connection failed:**
- Verify db container is healthy: `./docker-status.ps1`
- Check connection string format matches docker-compose.yml
- Verify credentials in .env match docker-compose.yml

**Frontend can't reach API:**
- Check API Gateway is running: `curl http://localhost:7080/health`
- Check proxy.conf.json points to correct address
- Check CORS configuration in services

For detailed troubleshooting, see **DOCKER_SETUP.md** section "Troubleshooting"
