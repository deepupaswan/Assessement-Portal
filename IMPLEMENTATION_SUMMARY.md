## 🔄 Database Changes

### SQL Server Containerization

**Why SQL Server:**
- ✅ Matches the existing SSMS-based workflow
- ✅ Keeps the database engine consistent with the current project
- ✅ Runs cleanly in a single shared container for local development
- ✅ Preserves the app's existing SQL Server EF Core provider path
- ✅ Easier migration path from the current codebase

**Connection String Changes:**
```
BEFORE: Server=(localdb)\MSSQLLocalDB;Database=X;Trusted_Connection=True;...
AFTER:  Host=service-db;Port=5432;Database=x_db;Username=x_user;Password=x_pw;...
```

**EF Core Provider Changes:**
```csharp
// BEFORE
options.UseSqlServer(connectionString)

// AFTER
options.UseNpgsql(connectionString)
```

### Package Dependencies to Add

In each service's `.csproj`, ensure:
```xml
<PackageReference Include="Microsoft.EntityFrameworkCore.SqlServer" Version="8.0.0" />
```

**Action Required**: Add this NuGet package to each service if not already present:
- [ ] Answer Service
- [ ] Assessment Service
- [ ] Candidate Service
- [ ] Identity Service
- [ ] Result Service

## 📨 Messaging Changes

### In-Memory → RabbitMQ

**Why RabbitMQ:**
- ✅ Distributed across containers
- ✅ Message persistence (survives restarts)
- ✅ Production-ready message broker
- ✅ Web management UI included
- ✅ Works identically to in-memory for development

**Configuration Changes:**
```csharp
// BEFORE: In-memory (development only)
x.UsingInMemory((context, cfg) => cfg.ConfigureEndpoints(context));

// AFTER: RabbitMQ (dev and production)
x.UsingRabbitMq((context, cfg) =>
{
    cfg.Host("rabbitmq", 5672, "/", h =>
    {
        h.Username("guest");
        h.Password("guest");
    });
    cfg.ConfigureEndpoints(context);
});
```

**RabbitMQ Access:**
- URL: http://localhost:15672
- Username: guest
- Password: guest

## 🐛 Common Issues & Solutions

### Issue 1: Port Already in Use
```powershell
# Find what's using port 7080
Get-NetTCPConnection -LocalPort 7080 -ErrorAction SilentlyContinue | 
  Select-Object OwningProcess, State

# Kill process (if safe)
Stop-Process -Id <ProcessId> -Force
```

### Issue 2: Container Won't Start
```powershell
./docker-logs.ps1 -Service api-gateway -Follow
# Check error messages and adjust configuration
```

### Issue 3: Database Migration Failed
```powershell
# Check database is running
./docker-status.ps1

# View migration errors
./docker-logs.ps1 -Service answer-service

# May need to delete volume and restart
./docker-down.ps1 -RemoveVolumes
./docker-build-all.ps1
./docker-up.ps1
```

### Issue 4: Services Can't Communicate
```powershell
# Check network
docker network inspect app-network

# Test connectivity
docker exec api-gateway ping identity-service

# Verify hostnames in connection strings
```
