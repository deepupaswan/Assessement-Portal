## Pre-Build Checklist

Before building and running containers, verify these prerequisites:

### System Requirements
- [ ] **Docker Desktop** installed (version 20.10+)
  ```powershell
  docker --version
  # Expected: Docker version 20.10 or higher
  
  docker compose version
  # Expected: Docker Compose version 2.0+
  ```

- [ ] **PowerShell 7+** installed (for helper scripts)
  ```powershell
  $PSVersionTable.PSVersion
  # Expected: 7.0 or higher
  ```

- [ ] **Available Disk Space**
  ```powershell
  # Need at least 10 GB free for:
  # - Docker images: ~4 GB
  # - Database volumes: ~2 GB
  # - RabbitMQ: ~100 MB
  ```

- [ ] **Available Ports** (not already in use)
  ```powershell
  # Check if ports are available
  Get-NetTCPConnection -LocalPort 4200 -ErrorAction SilentlyContinue   # Frontend
  Get-NetTCPConnection -LocalPort 7080 -ErrorAction SilentlyContinue   # API Gateway
  Get-NetTCPConnection -LocalPort 5672 -ErrorAction SilentlyContinue   # RabbitMQ
  Get-NetTCPConnection -LocalPort 15672 -ErrorAction SilentlyContinue  # RabbitMQ Mgmt
  ```

### File Structure Verification
- [ ] `.dockerignore` exists in root
- [ ] `Dockerfile` exists in each service:

  - [ ] `backend/assessment-service/Api/Dockerfile`
  - [ ] `backend/candidate-service/Api/Dockerfile`
  - [ ] `backend/identity-service/Api/Dockerfile`
  - [ ] `backend/result-service/Api/Dockerfile`
- [ ] `frontend/Dockerfile` exists
- [ ] `frontend/nginx.conf` exists
- [ ] `frontend/nginx-default.conf` exists
- [ ] `docker-compose.yml` exists
- [ ] `.env` file exists

### Configuration Files Verification
- [ ] All `appsettings.json` files contain SQL Server connection strings
- [ ] All `appsettings.Development.json` files contain environment variable placeholders
- [ ] `frontend/proxy.conf.json` points to `http://api-gateway:80`

## Functional Testing Checklist

### 1. Database Connectivity
```powershell
# Test SQL Server
docker exec sqlserver /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P SqlPassword123! -Q "SELECT @@VERSION"
```

- [ ] SQL Server returns version information

### 2. API Gateway Health Check
```powershell
curl http://localhost:7080/health
# Expected: {"status":"Healthy","service":"api-gateway"}
```

- [ ] Returns `200 OK`
- [ ] Response contains `"status":"Healthy"`

### 3. Service-to-Service Communication
```powershell
# Test from one service to another
docker exec api-gateway curl http://identity-service/health
docker exec api-gateway curl http://answer-service/health
docker exec api-gateway curl http://assessment-service/health
docker exec api-gateway curl http://candidate-service/health
docker exec api-gateway curl http://result-service/health
```

- [ ] All services are reachable from API Gateway
- [ ] All health checks return `200 OK`

### 4. RabbitMQ Connectivity
```powershell
# Check RabbitMQ status
docker exec rabbitmq rabbitmq-diagnostics -q ping
# Expected: ok

# Check from a service
docker exec answer-service curl -i -u guest:guest http://rabbitmq:15672/api/vhosts
# Expected: HTTP/1.1 200 OK
```

- [ ] RabbitMQ is responding to health checks
- [ ] Services can authenticate to RabbitMQ

### 5. Frontend Load Test
```powershell
# Test frontend is accessible

# Test frontend health endpoint
curl http://localhost:4200/health
# Expected: HTTP/1.1 200 OK with "healthy"
```

- [ ] Frontend loads without errors
- [ ] Nginx is serving static files

### 6. Frontend API Connectivity
Open browser to http://localhost:4200 and:
- [ ] Page loads without console errors
- [ ] No CORS errors in browser console
- [ ] Can make API calls to backend
- [ ] API responses are received

**Developer Tools Check:**

## Issue Resolution Checklist

If any checks fail:

### Problem: Container Won't Start
```powershell
./docker-logs.ps1 -Service api-gateway -Tail 50
# Read error message carefully
```
- [ ] Check if port is already in use
- [ ] Check if SQL Server migration failed
- [ ] Verify environment variables in `.env`

### Problem: Database Connection Failed
```powershell
./docker-logs.ps1 -Service answer-service -Follow
# Look for SQL Server connection errors
```
- [ ] Verify SQL Server container is healthy: `./docker-status.ps1`
- [ ] Check connection string format (should match docker-compose.yml)
- [ ] Verify credentials in `.env` match those in docker-compose.yml

### Problem: Services Can't Communicate
```powershell
docker exec api-gateway ping identity-service
# Should get responses (not "Name or service not known")
```
- [ ] Verify all containers on same network: `docker network inspect app-network`
- [ ] Check service names in connection strings/URLs
- [ ] Verify firewall isn't blocking container communication

### Problem: Frontend Can't Reach API
```
Open browser → F12 → Console → Network tab
Look for CORS errors or failed API requests
```
- [ ] Check proxy.conf.json points to correct gateway
- [ ] Verify CORS configuration in services allows frontend origin
- [ ] Check if API Gateway is actually running and healthy

### Problem: RabbitMQ Connection Failed
```powershell
./docker-logs.ps1 -Service rabbitmq -Follow
docker logs rabbitmq
```
- [ ] Verify RabbitMQ is healthy: `./docker-status.ps1`
- [ ] Check hostname: should be `rabbitmq` (not `localhost` or `127.0.0.1`)
- [ ] Verify credentials: guest/guest by default
